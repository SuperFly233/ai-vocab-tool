# Changelog

## v0.10.6 - 2026-07-14

- 历史 JSON 导入新增文件选择和拖拽导入，读取文件后自动填入文本框并执行预检，不会跳过确认直接合并。
- 设置页桌面端改为更明确的左说明、右控件宽屏布局；API 配置条、Prompt 工作区、导入区和标签管理会铺满可用宽度，减少右侧大块留白。

## v0.10.5 - 2026-07-14

- 设置页数据区新增历史 JSON 导入：支持粘贴普通历史数组、`{"raw":"[...]"}` 或 Supabase `value.raw` 格式。
- 导入前会预检新增、重叠、可更新、完全相同和导入文件内部重复数量；确认后才合并导入，不会替换或清空当前历史。
- 合并导入复用历史同步合并逻辑，会保留现有记录并补充不重叠记录、版本、追问、收藏、标签和备注。

## v0.10.4 - 2026-07-14

- 移除历史保存、查询保存和云同步合并里的 120 条截断，历史总数会按真实存储数量显示，不再长期卡在 120 条。
- 历史列表仍保留分批渲染和“继续显示”加载，避免大量记录一次性渲染导致移动端滚动卡顿。

## v0.10.3 - 2026-07-14

- 修复可视化编辑“常驻显示”开关被云同步合并逻辑粘住的问题；关闭后再次进入编辑页会恢复字段聚焦气泡，不再自动变回常驻显示。
- 设置布尔项现在会明确识别 `true` / `false` / `"true"` / `"false"` / `1` / `0`，避免字符串 `"false"` 被浏览器状态或云端数据误判为开启。

## v0.10.2 - 2026-07-14

- 恢复历史详情弹窗顶部标题、时间和操作按钮的原有展示方式，可视化编辑和 JSON 编辑时也不再隐藏顶部信息。
- 取消滚动后额外出现的压缩信息浮块，避免它在编辑区域中部重复展示词条摘要并挤占空间。

## v0.10.1 - 2026-07-14

- 可视化编辑改为字段级填写提示：聚焦具体输入项时弹出对应规则，气泡可用右上角关闭按钮临时隐藏，下次聚焦仍会出现。
- 设置页新增“可视化填写提示常驻显示”，开启后每个字段下方固定展示填写规则。
- 历史搜索框右侧新增搜索范围菜单，可选择只搜单词、释义、例句、搭配、辨析或其他相关项，避免只想搜词条时被例句命中。
- 可视化编辑和 JSON 编辑模式隐藏顶部学习信息块；首页查询输入框点击不再因为冒泡而自动回到顶部导致选中态失效。
- API 配置菜单打开时挂到 body 作为真正 fixed 浮层，更贴近当前配置条；配置和可视化条目改为拖拽排序，并带有插入位置反馈和边缘滚动。
- 模型 Prompt 和 API 设置区域继续放宽，避免右侧留下大块空白；修复向下弹出的 tooltip 箭头边框方向。

## v0.10.0 - 2026-07-14

- 版本策略从无限 `0.9.x` patch 滚动调整为语义化节奏：单点修复走 patch，成组体验或结构升级走 minor，核心路径稳定后再进入 `1.0.0`。
- 撤回历史筛选区误加的创建/修改日期范围输入，历史排序改成明确的“修改”和“创建”两个时间维度。
- 历史时间显示只在设置里做全局偏好；创建和修改时间一致时只显示一个裸时间，真正编辑过才按偏好显示创建、修改或全部。
- 修复较长 `translationHighlights` 被过滤掉的问题，译文里像“开发一款人工智能词汇工具”这样的整段对应项可以完整高亮。
- 可视化编辑补齐 `exampleHighlights` / `translationHighlights`、语义感受和近义/易混词字段，并支持义项、固定搭配和易混词上移下移或拖拽排序。
- 历史详情在可视化编辑和 JSON 编辑模式下隐藏学习用的吸顶信息栏，把空间还给编辑区；重新生成按钮改成紧凑刷新图标。
- 设置页大模块支持折叠展开，模型 Prompt 工作区撑满可用宽度；API 配置下拉固定贴近当前配置条，并支持配置顺序上下调整。

## v0.9.53 - 2026-07-14

- 修复重复排队去重里调用了不存在函数导致非空查询无反应的问题；查询入口现在会把异常明确 toast 出来。
- 历史列表和详情页只有在修改时间真正晚于创建时间时才显示“已编辑”，创建/修改相同的记录只显示一个时间。
- 设置页新增“历史时间显示”，可默认显示创建时间、修改时间或全部时间；详情弹窗顶部也可直接切换。
- 历史页新增创建日期和修改日期范围筛选。
- 历史详情滚动时保留压缩吸顶信息条，版本切换和重新生成按钮会缩小但不消失；API 配置弹窗有草稿时点击遮罩不会误关。

## v0.9.52 - 2026-07-14

- 设置页的模型 Prompt 专区改成“实现路径 + 编辑工作区”，展示输入、前端请求、system prompt、模型 JSON 和校验渲染的前后逻辑。
- Prompt 编辑器改为更宽的工作台布局，并显示当前来源、默认长度和编辑状态，减少窄高输入框和大片空白。
- 查询队列会识别正在执行或已经排队的相同请求，重复点击搜索不会继续堆出多条相同队列。
- 高亮逻辑支持 `~` / `～` 形式的搭配占位符，会把词条里的有效片段用于例句高亮，例如 `つもり～ということだ` 可命中例句里的 `つもり`。

## v0.9.51 - 2026-07-14

- 设置页新增模型 Prompt 专区，可查看默认查询 prompt、复制当前内容、保存自定义覆盖版或恢复默认。
- 自定义 prompt 会随 settings 云端同步；留空时继续使用内置默认 prompt，避免误改后影响查询。
- 查询请求会把自定义 system prompt 发送给 `/api/analyze`，默认 prompt 仍由后端统一维护并通过 `/api/config` 暴露给前端查看。

## v0.9.50 - 2026-07-14

- 例句高亮继续只围绕查询词本身及模型给出的实际词形，不再高亮上下文里的其它内容。
- 译文行会高亮当前义项或搭配对应的中文表达，例如“除了”“除此之外”“完全撇开 / 不谈”，但不会把成本、可行等上下文词一起标出来。
- 新增更明确的 `exampleHighlights` / `translationHighlights` 输出约定，用来兼容英语、中文、法语、德语等不同语言里的词形或译文对应变化。

## v0.9.49 - 2026-07-14

- 左右布局改为真正的宽屏工作台：左侧用于长输入和侧重点，右侧保留结果阅读空间，避免只是把顶部布局硬拆成两栏。
- 左侧输入框不再被固定高度卡死，可以拖拽调整；查询按钮移到输入控制区底部，整体操作顺序更自然。
- 侧重点在左右布局里只保留一个字段，不再出现两个旧输入框并排挤在底部的古早样式。

## v0.9.48 - 2026-07-14

- 结果高亮只使用当前查询词条本身及其常见英文词形变化，不再把义标、语意或模型给出的杂项词组一起高亮。
- 高亮只出现在例句和译文行，语意说明保持纯文本，避免阅读时出现一片零散标记。
- 首页标题和查询区滚动时保持置顶，并在滚动后自动压缩高度；点击查询输入区会回到顶部并聚焦输入。

## v0.9.47 - 2026-07-14

- 英文高亮会识别常见屈折变化，像 `transpire` / `transpired` / `transpiring` 会作为完整单词一起包住，不再只高亮半截词干。
- 义项和搭配支持可选 `highlightTerms`，模型可以把译文里实际对应的中文表达写进 JSON，避免 `shortestLabel` 与译文措辞不一致时漏高亮。
- 结果渲染会合并词条、义标、语意和 `highlightTerms` 做去重匹配，保留旧历史的兼容兜底。

## v0.9.46 - 2026-07-14

- 当模型供应商返回“正在使用 IPv6 访问”一类 HTML 拦截页时，服务端会自动把 `analyze`、`followup`、`models` 和 `test-profile` 请求转发到配置的 IPv4 relay。
- Cloudflare 版可把 `AI_IPV4_RELAY_BASE_URL` 指向 Vercel 版域名，浏览器仍然全程访问 Cloudflare，但模型请求可借用 Vercel 的 IPv4 出口。
- `/api/config` 会返回 `hasIpv4Relay`，便于确认当前部署是否启用了 IPv4 relay。

## v0.9.45 - 2026-07-14

- 新增 Cloudflare Pages Functions 适配层，复用现有 Vercel API handler 覆盖 `analyze`、`followup`、`sync`、`models`、`test-profile` 和 `config`。
- 新增 `build:pages` 构建脚本，只把公开静态资源复制到 `dist`，避免 Cloudflare Pages 直接公开仓库辅助文件。
- 补充 `wrangler.toml` 与 `package.json`，Cloudflare 版可继续使用同一个 Supabase `study_store` 数据库和同源 `/api/*` 调用。

## v0.9.44 - 2026-07-14

- 查询进行中清空主输入框不再触发 `resetCurrentLookupState()`，因此不会意外取消当前流式查询或清空等待队列。
- 队列里的下一条开始执行时会同步回主输入、方向和备注字段，用户能看清当前正在处理哪条。
- 查询忙碌时遇到已有历史记录会直接加入队列并保留 `existingId`，不再弹确认框打断连续排队。

## v0.9.43 - 2026-07-13

- 取消主查询流式预览里的块级上浮动画，避免模型 delta 频繁刷新时所有区块反复浮动。
- 追问生成中的正文也不再反复触发入场动画，仅保留轻量光标和 pending 状态，优先保证阅读稳定。

## v0.9.42 - 2026-07-13

- 历史记录手机端默认收起高级筛选，只保留范围、搜索、筛选摘要和横向排序条，首屏能更快看到正式条目。
- 移动端历史条目改成内容优先、操作按钮置底，避免右侧按钮栏把词条、摘要和标签挤成很窄的一列。
- 历史列表改为分批渲染并在接近底部时继续加载，减少 120 条记录同时渲染带来的滚动卡顿；主查询和追问流式内容增加透明度与轻微上浮缓动。

## v0.9.41 - 2026-07-13

- 主查询请求现在会向 `/api/analyze` 发送 `stream:true`，服务端按 OpenAI-style SSE 转发模型 delta，不再等完整 JSON 才返回。
- 前端会边接收边显示原始 JSON，并从已生成文本中提取 headword、义项、搭配、语感和易混块做实时预览。
- 模型结束后服务端仍会执行完整 JSON 校验和必要修复，历史保存只使用最终校验后的结构化结果。

## v0.9.40 - 2026-07-13

- 追问回答的普通正文在手机端进一步收紧字号和行高，避免比表格内容显得更大。
- Markdown 表格现在被限制在自己的横向滚动容器内，不再把整段追问回答撑成一个可左右拖动的大块。
- 表格单元格里的 `<br>` 会渲染为真正换行，不再把标签文本显示出来。

## v0.9.39 - 2026-07-13

- API 配置菜单改为 fixed 顶层浮层，并在移动端避开底部导航，修复被主题、布局或导航按钮压住的问题。
- 主查询等待期改为先展示词条头、义项、搭配、语感、易混和相关记录等骨架块，右上角显示估算百分比。
- 真实结果返回后会按结构块渐变接管并逐字显示内容，JSON 页也会逐段逐字揭示最终校验后的 JSON。

## v0.9.38 - 2026-07-13

- 主查询结果打字机升级为结构分块渲染，按词条头、义项、搭配、语感、易混和相关记录等块依次出现。
- 每个结构块会先淡入，再逐字显示块内文本，避免整页空框或整段一次性出现。
- 继续保留完整 JSON 校验后再渲染的策略，避免半截 JSON 破坏词条结构或历史保存。

## v0.9.37 - 2026-07-13

- API 配置外层只保留当前配置名称和下拉箭头，新增、选择、编辑、删除都收进下拉菜单，减少设置页按钮堆叠。
- 新增/编辑 API 配置弹窗里新增“测试连接”，保存前即可验证当前填写的 URL、Key 和 Model，不会先写入本地设置。
- 词条结果顶部把核心义单独突出为一行，类型、词性和方向合并为同一行辅助信息。
- 主查询结果在 JSON 校验完成后会逐字揭示结构化词条内容；手机端追问正文缩小并收紧行高。

## v0.9.36 - 2026-07-13

- 设置页当前 API 配置卡片新增“测试连接”，可用极小请求验证 URL、Key、Model 或环境变量配置是否可用。
- 新增 `/api/test-profile` 端点，复用管理员环境变量权限校验，并返回连接耗时与明确失败原因。
- 当前配置卡片会显示尚未测试、测试中、连接正常或连接失败状态，减少配置后不知道是否生效的盲区。

## v0.9.35 - 2026-07-13

- 追问回答优先使用 SSE 流式返回，模型生成时会实时更新正在生成的回答卡片，减少等待时的空白感。
- `/api/followup` 兼容 OpenAI-style stream 数据，并保留普通 JSON 返回路径，不支持流的接口仍可给出正常结果或明确错误。
- 生成中的追问卡片新增轻量光标动效，最终回答仍会保存到对应历史记录并随云端同步。

## v0.9.34 - 2026-07-13

- 追问回答里的 Markdown 表格按列数使用更稳的最小列宽，手机上多列表格会横向滚动，不再把每列压得过扁。
- 表格容器限制在回答区域内，支持触摸惯性滚动，少列表格保持铺满，多列表格保持可读。
- 补齐窄屏样式，避免表格撑破页面或在历史详情里挤压其它内容。

## v0.9.33 - 2026-07-13

- 设置页“数据”区域新增标签管理，可汇总查看所有 Tag 及对应历史数量。
- 标签可一键跳转到历史筛选，也支持批量重命名和从全部历史中移除。
- 重命名和移除会更新相关历史记录并随云端同步，补齐 Tag 整理闭环。

## v0.9.32 - 2026-07-13

- 配置、权限、JSON 格式等不可恢复错误不会盲目重试，减少无意义 API 消耗。

## v0.9.31 - 2026-07-13

- 队列支持上移、下移、插队到最前和移除，当前查询完成后会自动执行下一条。
- 队列会保留提交时的语言方向和侧重点，清空输入时也会同步清空等待队列。

## v0.9.30 - 2026-07-13

- 字体偏好写入 settings 并随云端同步，多设备会保持一致。
- 字体切换通过全站 CSS 变量生效，JSON、代码和密钥字段继续使用等宽字体。

## v0.9.29 - 2026-07-13

- 可视化编辑保存时会同步更新 JSON、历史标题与预览，原始 JSON 编辑模式仍完整保留。
- 移动端追问表格会按列数调整单列宽度，避免多列表格被压扁或横向过宽。

## v0.9.24 - 2026-07-13
